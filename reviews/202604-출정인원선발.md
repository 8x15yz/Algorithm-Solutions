## 문제 설명

체력·정신력·기술력·방어력 총합을 기준으로 상위 30%의 기술자를 선발한다.

- `limit = floor(n × 0.3)`이 0이면 아무도 선발하지 않는다.
- 선발 경계(rank `limit`)에 동점자가 있으면 해당 점수를 가진 사람을 전원 제외한다.
- 남은 선발자가 없으면 빈 리스트를 반환한다.
- 결과는 알파벳 역순으로 출력한다.

### 입출력 예

| 입력 | 출력 |
|------|------|
| `[['A',25,24,11,12],['B',13,22,16,14]]` | `[]` |
| `[['A',25,25,25,25],['B',10,12,13,11],['C',24,22,23,21],['D',13,22,16,14]]` | `['A']` |

**예시 설명**  
1번: 2명 중 1명 선발 = 50% > 30% → 아무도 선발 불가  
2번: 4명 중 1명(25% < 30%) → 최고점 A(100점)만 선발

## 원본 코드

```python
def solution(data):
    log = ""
    res = []
    for arr in data:
        sum_skill = sum(arr[1:])
        if res == []:
            res += [(arr[0], sum_skill)]
        if sum_skill >= res[0][1] : 
            if (arr[0], sum_skill) not in res:
            	res = [(arr[0], sum_skill)] + res
        else:
            res += [(arr[0], sum_skill)]
    party = len(res)
    party_30 = int(party*0.3)
    final = res[:party_30]
    while final:
        if res[party_30][1] == final[-1][1]:
            final.pop()
        else:
            break
    f_res = []
    if final:
        for fin in final:
            f_res += fin[0]

    return f_res
```

## 01 정확성 ⚠️

**결과**: 정렬 버그·역순 정렬 누락·문자 분리 버그 3가지 발견 (제공 예시는 우연히 통과)

### ① 정렬 로직 버그

`res[0]`(현재 최고점)과만 비교하므로, 현재 최고점보다 낮은 새 원소는 무조건 맨 뒤에 추가된다.
중간 점수의 원소가 올바른 위치에 삽입되지 않아 `res`가 내림차순 정렬 상태가 되지 않는다.

```
입력: [A=100, B=46, C=90, D=65]
res: [('A',100), ('B',46), ('C',90), ('D',65)]  ← C와 B 순서 뒤집힘
```

예시 2번은 A(최고점)가 첫 번째 입력이라 `res[0]`이 항상 최고점으로 유지되어 통과하지만,
A가 중간에 입력되면 오답이 된다.

### ② 알파벳 역순 정렬 누락

결과를 알파벳 역순으로 출력해야 하지만 `f_res`에 대한 정렬 로직이 없다.
제공 예시는 선발자가 1명이라 순서 버그가 드러나지 않는다.

### ③ `f_res += fin[0]` 문자 분리 버그

`fin[0]`은 문자열이므로 리스트에 `+=`하면 각 문자가 개별 원소로 추가된다.
이름이 한 글자인 예시에서는 우연히 동작하지만, 여러 글자 이름에서 깨진다.

```python
# 버그
f_res += fin[0]          # 'Alice' → ['A','l','i','c','e']
# 수정
f_res.append(fin[0])     # 'Alice' → ['Alice']
```

### ④ 미사용 변수

`log = ""`가 전혀 사용되지 않는다.

### 개선 코드

```python
def solution(data):
    scored = sorted(
        [(arr[0], sum(arr[1:])) for arr in data],
        key=lambda x: x[1],
        reverse=True
    )

    n = len(scored)
    limit = int(n * 0.3)

    if limit == 0:
        return []

    # ① limit으로 자른다 — 경계 마지막 사람의 점수를 기준점으로 저장
    cutoff = scored[limit - 1][1]

    # ② 경계(scored[limit-1])와 바로 다음(scored[limit])이 동점이면
    #    그 점수를 경계 안에서도 전원 제거
    if limit < n and scored[limit][1] == cutoff:
        selected = [name for name, score in scored[:limit] if score > cutoff]
    else:
        selected = [name for name, _ in scored[:limit]]

    return sorted(selected, reverse=True)
```

## 02 시간복잡도 ⚠️

**결과**: O(n²) — `sorted()` 활용으로 O(n log n)으로 개선 가능

- 루프 내 `not in res`: 리스트 선형 탐색 O(n)
- `res = [x] + res`: 리스트 앞 삽입 O(n)
- 전체: O(n²)

n ≤ 10이라 실질 차이는 없지만, `sorted()` 한 번으로 O(n log n)에 완전 정렬 가능.
