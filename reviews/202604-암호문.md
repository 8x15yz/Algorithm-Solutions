## 01 정확성

**⚠️ 문제 있음 — ValueError 발생 가능**

`data[a]`가 r/e/v이고 다음 문자가 r/e/v도 아니고 숫자도 아닐 때 `else` 브랜치에서 `int(data[a+1:a+2])`가 터진다.

예: `"reverse"` 문자열에서 `'r'` 다음 `'s'`가 오면 `int('s')` → **ValueError**

또한 `'' in 'rev'`가 `True`를 반환한다는 점도 주의 필요 — 문자열 끝에서 빈 슬라이스가 continue로 빠지는 건 우연히 맞지만 의도가 불명확하다.

```python
# 수정: 다음 문자가 숫자가 아니면 건너뜀
if not next_one.isdigit():
    continue
```

## 02 시간복잡도

**✅ 이상 없음 — O(n)**

단일 루프, 내부 연산은 모두 고정 크기 슬라이싱 O(1). 중첩 루프 없음.

다만 `data[a+1:a+2]`와 `data[a+1:a+3]`을 매 반복마다 중복 생성 — 변수로 한 번만 뽑으면 된다.

---

## 03 공간복잡도

**✅ 이상 없음 — O(1)**

결과 누적 변수 하나, 임시 슬라이스 몇 개 외에 추가 자료구조 없음. 재귀도 없음.

## 04 코드 품질

**⚠️ 문제 있음 — 3가지**

**① 변수명이 의도를 숨긴다**
- `a` → `i` 또는 `idx`
- `res` → `date_sum` 또는 `total`

**② `data[a] in 'rev'` 패턴이 오해를 부른다**

`'rev'`는 문자열이라 `''`도 포함하고, `'re'`도 포함한다. 집합으로 의도를 명시해야 한다.

```python
data[i] in {'r', 'e', 'v'}   # 명확
```

**③ 슬라이스 중복 + 조건 구조 개선 여지**

```python
def solution(data):
    total = 0
    for i in range(len(data)):
        if data[i] not in {'r', 'e', 'v'}:
            continue
        one = data[i+1:i+2]
        if not one.isdigit():
            continue
        two = data[i+1:i+3]
        if two.isdigit() and int(two) <= 10:
            total += int(two)
        else:
            total += int(one)
    s = str(total)
    return s[0] + '월 ' + s[1:] + '일'
```
