셀프 코드리뷰 체크리스트
01 정확성 엣지케이스 · 제약조건
  • 빈 문자열, 빈 리스트, 최솟값/최댓값 입력을 통과하는가
  • 인덱스 범위를 벗어나는 경우가 없는가
  • 문제의 제약조건을 코드가 전부 반영하는가
02 시간복잡도 빅오 · 중첩루프
  • 내 풀이의 빅오는 무엇인가? (말로 써보기)
  • 중첩 루프가 있다면 O(n²) 이상 아닌가
  • 리스트의 in / remove 를 set/dict 으로 대체할 수 있는가 (O(n) → O(1))
  • 완전탐색 → BFS/DFS → DP 순으로 개선 가능한가
03 공간복잡도 메모리 · 재귀깊이
  • 불필요하게 리스트/딕셔너리를 복사하고 있진 않은가
  • 결과를 한 번에 다 올리지 않고 제너레이터로 대체 가능한가
  • 재귀를 쓴다면 최대 깊이가 안전한가 (기본 1000, sys.setrecursionlimit 필요 여부)
04 코드 품질 가독성 · 변수명
  • 변수명이 의도를 설명하는가 (a, b, tmp → count, target, current_sum)
  • 같은 로직이 두 번 이상 반복되면 함수로 뺐는가
  • 주석 없이도 읽히는가 — 안 읽히면 구조/변수명 문제
  • 조건문이 3단계 이상 중첩되진 않는가
05 보안 관점 입력검증 · ReDoS
  • 외부 입력 크기를 제한하고 있는가
  • 정규표현식을 쓴다면 ReDoS 위험은 없는가 ((a+)+$ 형태 주의)
  • 민감한 값을 == 로 직접 비교하고 있진 않은가 (hmac.compare_digest 권장)
