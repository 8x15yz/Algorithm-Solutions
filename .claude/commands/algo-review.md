---

**리뷰 대상 코드:**

$ARGUMENTS

---

## 리뷰 기준

### 01 정확성
- 엣지케이스 (빈값, 최솟값, 최댓값, 음수, 0)를 처리하는가
- 인덱스 범위를 벗어나는 경우가 없는가
- 문제의 제약조건이 코드에 전부 반영됐는가

### 02 시간복잡도
- 이 코드의 빅오 표기법은 무엇인가 (명시해줘)
- 중첩 루프가 있다면 O(n²) 이상인지 확인
- 리스트의 `in` / `remove` / `insert` 를 `set` / `dict` 으로 개선할 수 있는가
- 더 낮은 복잡도로 풀 수 있는 알고리즘이 있는가 (완전탐색 → BFS/DFS → DP)

### 03 공간복잡도
- 불필요하게 리스트/딕셔너리를 복사하고 있는가
- 제너레이터로 대체할 수 있는 부분이 있는가
- 재귀를 쓴다면 최대 깊이가 안전한가 (파이썬 기본 1000)

### 04 코드 품질
- 변수명이 의도를 설명하는가 (`a`, `b`, `tmp` → `count`, `target`, `current_sum`)
- 중복 로직이 있다면 함수로 뺄 수 있는가
- 조건문 중첩이 3단계 이상이지 않은가
- 주석 없이도 읽히는가

### 05 보안 관점 (선택)
- 외부 입력이 있다면 크기 제한을 두고 있는가
- 정규표현식이 있다면 ReDoS 위험한 패턴은 아닌가
- 민감한 값 비교에 `==` 를 직접 쓰고 있진 않은가
